ML-Based Crop Yield Forecasting Under Climate Change

Tech: Python, scikit-learn, XGBoost, TensorFlow/Keras, Machine Learning (Random Forest, LSTM), Time-Series Analysis

Project Summary

This project addresses the critical challenge of ensuring global food security in the face of climate change. As part of a Kaggle competition, I developed and evaluated a suite of machine learning models—including **Random Forest, XGBoost, and Long Short-Term Memory (LSTM) networks**—to predict future crop viability. By integrating historical climate data, soil properties, and crop yield records, the models learned to forecast yields for wheat and maize. The analysis identified key environmental drivers, such as **soil nitrogen and short-wave radiation**, and provided a scalable framework for adapting agricultural strategies to climate uncertainty.

The Business Problem

Climate change poses a significant risk to global agriculture through shifting weather patterns and extreme events. For governments, agricultural organizations, and farmers, the ability to accurately forecast crop yields under future climate scenarios is essential for strategic planning, resource allocation, and mitigating food shortages. This project aimed to answer the question: **Can we build reliable machine learning models to predict how crop yields will change in different locations over time?** The goal was to create a data-driven tool that could help stakeholders make proactive decisions to ensure a stable and secure food supply.

XGBoost, a powerful tree-based model, was used for its high performance and interpretability on the tabular dataset of static and aggregated climate features.

2. Building a Time-Series Model (LSTM)

An LSTM network was developed to capture temporal dependencies in the sequential climate data (e.g., daily temperature and precipitation). This deep learning approach is ideal for learning from time-series patterns.

from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense, Input

# Define the LSTM model architecture
model = Sequential([
    Input(shape=(timesteps, n_features)),
    LSTM(units=64, return_sequences=True),
    LSTM(units=32),
    Dense(units=1) # Output layer for yield prediction
])

model.compile(optimizer='adam', loss='mean_squared_error')

# Train the model
history = model.fit(X_train_ts, y_train, epochs=50, batch_size=32, validation_split=0.2)

Challenges

  • Feature Engineering vs. Raw Data: A key challenge was determining the right level of feature engineering. While initial intuition suggested creating abstract features (e.g., "heat stress days"), the final analysis showed that the LSTM model performed significantly better when fed the raw, high-resolution climate data directly, allowing it to learn more complex temporal patterns on its own.
  • Data Fusion: The project required integrating two distinct types of data: static, location-based soil data and dynamic, time-series climate data. Developing a model architecture that could effectively combine these different data structures was a significant technical challenge.
  • Model Selection and Interpretability: A major part of the project involved comparing the performance of traditional models (Random Forest, XGBoost) against a deep learning model (LSTM). While the LSTM ultimately performed well, the tree-based models provided greater interpretability, offering clearer insights into which specific features were the most important predictors of crop yield.
